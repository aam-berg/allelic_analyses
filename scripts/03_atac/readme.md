# `03_atac/` — F121-9 ATAC-seq Processing Pipeline

Processes raw paired-end ATAC-seq from GSE198517 (4 replicates of F121-9
mESCs, untreated) into:

1. **Consensus peaks** — used by `05_motif_annot/` for the per-motif "overlaps
   an ATAC peak" annotation.
2. **Allele-specific BAMs and bigWigs** — used by `06_allele_pairs/` to
   quantify allele-specific accessibility at and near motifs.

## Input

GEO series GSE198517, samples GSM5949801–GSM5949804, SRRs:

| Replicate    | GSM         | SRR          |
|--------------|-------------|--------------|
| rep1         | GSM5949801  | SRR18313162  |
| rep2         | GSM5949802  | SRR18313161  |
| rep3         | GSM5949803  | SRR18313160  |
| rep4         | GSM5949804  | SRR18313159  |

The canonical sample-to-SRR mapping is in `samples.tsv`. To process a
different ATAC dataset, edit `samples.tsv` and re-run; nothing else needs
changing.

## Pipeline steps

| Step  | Script                          | What it does                                        | Mode             |
|-------|---------------------------------|-----------------------------------------------------|------------------|
| 00    | `00_setup_references.sh`        | mm39 bowtie2 index (reuses 04_proseq's if present)  | single           |
| 01    | `01_download_fastq.sh`          | fasterq-dump per sample, R1 + R2, FastQC raw        | array (4)        |
| 02    | `02_trim_qc.sh`                 | cutadapt PE trim, FastQC trimmed                    | array (4)        |
| 03    | `03_align_filter.sh`            | bowtie2 PE → MAPQ → chrM → dedup                    | array (4)        |
| 04    | `04_call_peaks.sh per-rep`      | MACS2 per-replicate peaks                           | array (4)        |
| 04    | `04_call_peaks.sh consensus`    | Consensus across reps                               | single           |
| 05    | `05_wasp_setup.sh`              | Clone WASP, build SNP files                         | single           |
| 06    | `06_wasp_filter.sh`             | WASP mapping bias removal (PE)                      | array (4)        |
| 07    | `07_allele_specific.sh`         | Split BAMs by allele (lib/split_by_allele.py)       | array (4)        |
| 08    | `08_make_bigwigs.sh`            | Per-rep + allele-specific bigWigs                   | array (4)        |
| 09    | `09_merge_qc.sh`                | Merged bigWigs + final QC report                    | single           |

## Running the pipeline

**Dry run (default):**
```bash
cd 03_atac
bash run_pipeline.sh
```

**Submit:**
```bash
bash run_pipeline.sh --submit
```

The orchestrator submits all steps with proper SLURM dependencies. You can
also run individual steps manually for debugging:

```bash
sbatch --array=0-3 --partition=short --time=12:00:00 --mem=32G \
       --cpus-per-task=8 -o logs/03_%A_%a.out -e logs/03_%A_%a.err \
       03_align_filter.sh
```

Every step is **idempotent**: it checks for its expected output and skips if
present. To force a re-run, delete the relevant output files first.

## Key design decisions

**MAPQ ≥ 30.** ATAC standard, distinct from PRO-seq's MAPQ ≥ 10. Higher
because PE alignment provides more positional information.

**chrM removal before everything else.** ATAC libraries can have 30–60% chrM
reads (mitochondrial genome is naked DNA, highly accessible to Tn5). Removed
right after MAPQ filter.

**Duplicate removal.** Unlike PRO-seq, ATAC duplicates are real PCR duplicates
(not focal pause biology). Removed with `samtools markdup -r`. Standard.

**No spike-in, no rRNA filtering.** Neither applies to ATAC.

**Peak calling before WASP, not after.** Standard ATAC+WASP workflow. Peak
boundaries reflect biology, not allele balance, so they're called once on
filtered BAMs. WASP is then applied for allele-specific quantification within
those peaks (downstream, in `06_allele_pairs/`).

**MACS2 parameters.** `--nomodel --shift -75 --extsize 150 -f BAMPE -q 0.01
--keep-dup all`. The `--shift -75 --extsize 150` adjustment centers the
analyzed region on the Tn5 cut site for narrow-peak calling. `--keep-dup all`
because dedup is already done. `-f BAMPE` tells MACS2 to use fragment
midpoints from PE inserts.

**Consensus peaks: present in ≥2 of 4 replicates.** The threshold lives in
`config.sh` as `MIN_REPS_FOR_CONSENSUS`. 2/4 is a reasonable compromise
between sensitivity (1/4 = union) and stringency (4/4 = intersection).

**WASP for paired-end.** Uses `--is_paired_end` flag of
`find_intersecting_snps.py`. The remap step uses **identical** bowtie2
parameters as the original alignment (`--very-sensitive -X 1000
--no-mixed --no-discordant`). This is critical: WASP's logic only works
if mapping differences between original and remapped reads are due to the
allele, not to aligner settings.

**Allele splitting is fragment-aware.** `lib/split_by_allele.py
--paired-end` classifies each fragment (read pair) based on combined
SNP evidence from both mates. Both mates of a fragment go to the same
output BAM. This avoids the double-counting that would occur if R1 and R2
were classified independently.

**Tn5 shift not applied.** The +4/−5 Tn5 cut-site adjustment is useful for
footprinting but not needed for peak calling (MACS2 handles fragment-level
analysis with `-f BAMPE`) or for fragment-level coverage bigWigs. If
single-base cut-site precision is needed later (e.g., footprinting analyses
in `07_analysis/`), apply the shift on the BAMs at that point.

**Fragment-coverage bigWigs.** `bedtools genomecov -bg -pc` uses the full
fragment span (R1 leftmost to R2 rightmost). Standard for browser
visualization. **No strand split** — unlike PRO-seq, ATAC reads on both
strands at the same accessible region are equivalent.

## Cross-pipeline dependencies

This pipeline reads from:

- `01_resources/`: `mm39.fa`, `mm39.chrom.sizes`, `F121-9_het_snps.ucsc.vcf.gz`
- `lib/`: `make_wasp_snps.sh`, `split_by_allele.py`
- `04_proseq/`: the mm39 bowtie2 index, if already built (reused if present;
  built fresh if not)

Outputs consumed by:

- `05_motif_annot/02e_annot_atac_overlap.R`: reads `peaks/consensus/consensus_peaks.bed`
- `06_allele_pairs/03_atac_alleles_at_motifs.R`: reads
  `allele_specific/{SAMPLE}_{ref,alt}.bam` and `bigwig/allele_specific/*.bw`

## Troubleshooting

- **MACS2 not found.** `conda install -n proseq2_env -c bioconda macs2`
- **fasterq-dump fails on SRA.** Check disk space in `${SCRATCH_DIR}/tmp`.
- **WASP install error.** Make sure `pysam` is in the conda env;
  `05_wasp_setup.sh` will install it if missing.
- **Dedup removes most reads.** ATAC libraries with low complexity can have
  >50% duplicates. The pipeline reports the rate; if it's >70%, the library
  may be too low-input — flag and decide whether to proceed.
- **chrM fraction very high (>70%).** Indicates poor nuclei prep. Dataset
  may still be usable but with much-reduced effective depth.

## Future extensions
