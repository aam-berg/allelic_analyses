# `03_motif_annot/` — Per-motif genomic annotation

For every motif archetype found in `02_motif_scanning/`, annotate every
hit with rich genomic context: gene/promoter overlap, nearest-TSS distance
(strand-aware), genic region, ATAC overlap, F1 het-SNP overlap (in motif
and concentric flanking bins), and per-instance sequence metadata.

Inputs come from `01_resources/` (genome, VCFs) and `02_motif_scanning/`
(per-motif BED files in `merged/per_motif/`).

---

## Pipeline at a glance

```
01_preprocess_resources.R           (run ONCE — TxDb, TSS GRanges, ATAC liftOver)
            │
            ▼ produces preprocessed/*.rds, preprocessed/*.sqlite
            │
02_annotate_motifs.R                (run PER motif via SLURM array)
            │
            ▼ produces annot_motifs/<MOTIF>_annotated.tsv.gz
            │
0x_find_missing_motifs.sh           (utility — list motifs that didn't finish)
            │
            ▼ writes missing_motifs.txt
            │
0x_resubmit_missing_motifs.sh       (utility — retry missing list with bumped limits)
```

---

## Quickstart

```bash
# 1. Preprocess shared resources (TxDb, TSS, ATAC liftOver). Once.
sbatch 01_run_preprocess_resources.sh

# 2. Submit per-motif annotation array
bash 02_annotate_motifs.sh                  # dry run
bash 02_annotate_motifs.sh --apply

# 3. After step 2 finishes, sweep for missing outputs
bash 0x_find_missing_motifs.sh
bash 0x_resubmit_missing_motifs.sh --dry-run
bash 0x_resubmit_missing_motifs.sh

# 4. Re-sweep to confirm
bash 0x_find_missing_motifs.sh
```

A typical per-motif annotation completes in 5–30 min in 20 GB.

## Outputs

```
${OUTDIR}/
├── preprocessed/
│   ├── gencode_vM38_txdb.sqlite
│   ├── tx_metadata.rds
│   ├── tss_granges.rds
│   ├── tss_unique_granges.rds
│   ├── atac_peaks_<sample>.rds      # one per ATAC bigBed (mm39, lifted)
│   └── mm10ToMm39.over.chain[.gz]
├── motif_file_list.txt
├── run_annotate_array.sh            # generated by 02_annotate_motifs.sh
├── missing_motifs.txt               # generated by 0x_find_missing_motifs.sh
├── run_annotate_missing.sh          # generated by 0x_resubmit_missing_motifs.sh
├── <MOTIF>_annotated.tsv.gz         # one per motif archetype
└── logs/
```

Per-hit columns in each `<MOTIF>_annotated.tsv.gz`:

| Group | Columns |
|---|---|
| Hit identity | `chrom, start, end, motif_id, score, strand` |
| Sequence | `motif_sequence` (strand-aware), `palindromicity_score` (per-instance) |
| Gene context | `is_intragenic`, `intragenic_gene_types`, `genic_regions` |
| Promoter context | `is_promoter`, `promoter_transcript_ids`, `promoter_tsls`, `promoter_gene_names` |
| Strand-aware TSS | `upstream_tss_dist`, `upstream_tss_gene_names`, `upstream_tss_tx_ids`, `upstream_tss_tsls`; same for `downstream_…` |
| ATAC overlap | `atac_<sample>_overlap`, `…_peak_coords`, `…_signalValue`, `…_pValue`, `…_qValue` |
| Direct SNP overlap | `snp_<hybrid>_overlap`, `…_details`, `…_count` |
| Flanking SNP bins | `snp_<hybrid>_flank<D>bp_overlap/_details/_count` for each `D` in `--flank_distances` |

### Strand-aware TSS

"Upstream" and "downstream" are defined relative to the **motif's own
strand**, not raw genomic coordinates:
- `+` motif: upstream = lower genomic coord, downstream = higher
- `−` motif: upstream = higher genomic coord, downstream = lower

This matches the biological question: "what TSS is 5′ vs 3′ of this
binding site, relative to the orientation in which the TF is reading
the DNA?"

### Flanking SNP bins are non-overlapping concentric annuli

For `--flank_distances 10,25,50,100,200,400,800`:
- `flank_10bp`: positions 1–10 bp from motif edge
- `flank_25bp`: positions 11–25 bp (excludes 0–10)
- `flank_50bp`: positions 26–50 bp (excludes 0–25)
- … and so on.

The motif itself is excluded from all flank bins.

## Configuration

All paths and parameters are in `config.sh`. Every script in this
directory sources it:

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "${SCRIPT_DIR}/config.sh"
```

Key variables:

| Variable | Default | Purpose |
|---|---|---|
| `GENOME_FA` | `…/mm39.fa` | Indexed FASTA |
| `MOTIF_DIR` | `…/moods_scan/merged/per_motif` | Where step 2 of motif scanning wrote BEDs |
| `VCF_DIR` | `…/resources/vcf` | Per-hybrid het-SNP VCFs |
| `GTF_FILE` | `…/gencode.vM38…gtf.gz` | For TxDb construction |
| `CHAIN_FILE` | `…/mm10ToMm39.over.chain` | For ATAC liftOver |
| `ATAC_DIR` | `…/atac/m_musculus` | bigBed input dir |
| `ATAC_FILES` | `4DNFIAEQI3RP.bb,4DNFIZNPOOZN.bb` | comma-separated bigBed filenames |
| `OUTDIR` | `…/annot_motifs` | Pipeline output root |
| `HYBRIDS` | `F121-9,BL6xCAST` | comma-separated hybrid labels |
| `FLANK_DISTANCES` | `10,25,50,100,200,400,800` | comma-separated bin edges |
| `PROMOTER_UPSTREAM` | `1000` | bp upstream of TSS |
| `PROMOTER_DOWNSTREAM` | `500` | bp downstream of TSS |
| `DEFAULT_PARTITION/TIME/MEM` | `short / 01:00:00 / 20G` | Default array task |
| `RETRY_PARTITION/TIME/MEM` | `medium / 72:00:00 / 128G` | Retry array task |
| `CONDA_ENV` | `test_m` | R + Bioconductor |

**Promoter window**: keep `PROMOTER_UPSTREAM` and `PROMOTER_DOWNSTREAM`
identical to the values in `02_motif_scanning/config.sh`.

## Scripts

### Main pipeline

| Script | Type | Purpose |
|---|---|---|
| `01_preprocess_resources.R` | R | Build TxDb, write TSS GRanges, import + liftOver ATAC bigBeds. Run once. |
| `01_run_preprocess_resources.sh` | SLURM | Wrapper for step 1. |
| `02_annotate_motifs.R` | R | Annotate one motif's BED. |
| `02_annotate_motifs.sh` | bash | Build SLURM array script, submit. Skips already-annotated motifs (idempotent). |

### Recovery utilities

| Script | Purpose |
|---|---|
| `0x_find_missing_motifs.sh` | Compares input BEDs against `<MOTIF>_annotated.tsv.gz`. Writes missing/empty to `missing_motifs.txt`. |
| `0x_resubmit_missing_motifs.sh` | Reads that list and submits a retry array with bumped limits (default `RETRY_PARTITION=medium`, 72 h, 128 G). Removes any partial output first so the R script doesn't skip. |

### Archived

| Path | Status |
|---|---|
| `0____old/` | Archive of older script versions. Ignore. |

## Recovery flow

The default array runs in `short` with 1 h / 20 GB. Some motifs (very
common short PWMs with millions of hits) may time out or run out of
memory. After the main array job finishes:

```bash
bash 0x_find_missing_motifs.sh                     # cheap; just file checks
bash 0x_resubmit_missing_motifs.sh --dry-run       # preview retry
bash 0x_resubmit_missing_motifs.sh                 # retry in `medium`, 72h, 128G
bash 0x_find_missing_motifs.sh                     # confirm complete
```

If the second sweep still finds missing motifs, look in
`${OUTDIR}/logs/` — at that point the failure isn't a resource issue
and probably indicates a real bug (missing input, malformed VCF,
unexpected ATAC mcols, etc.).

## Where ATAC-seq enters the pipeline

| File | What's there |
|---|---|
| `config.sh` | `ATAC_DIR`, `ATAC_FILES` |
| `01_preprocess_resources.R` | **STEP 5**: bigBed import, mm10→mm39 liftOver, saves `atac_peaks_<sample>.rds`. Assumes `signalValue/pValue/qValue` mcols. |
| `02_annotate_motifs.R` | Step `[5/6]` (auto-discovers `atac_peaks_*.rds`); **ANNOTATION 8** (overlap + per-peak metadata). |

To swap ATAC datasets, see "Swapping ATAC datasets" below.

### Swapping ATAC datasets

1. Drop new `.bb` files into `${ATAC_DIR}`.
2. Update `ATAC_FILES` in `config.sh`.
3. Delete old `${PREPROCESSED_DIR}/atac_peaks_*.rds`.
4. Re-run `01_run_preprocess_resources.sh`.
5. Delete any `<MOTIF>_annotated.tsv.gz` you want re-annotated, then
   re-run `02_annotate_motifs.sh --apply`.

If your new bigBed lacks `signalValue/pValue/qValue` columns,
`02_annotate_motifs.R::ANNOTATION 8` will fail. Add a defensive check
or strip those columns from the assignment block.

## Known issues / things to watch

1. **`gene_metadata.rds`** is created by step 1 but never loaded by
   step 2. Either remove the save in step 1 or wire it in.

2. **`motif_strand` parameter in `fill_tss_info()`** is passed but
   never used inside the function body (cosmetic).

3. **ATAC bigBed metadata columns assumed**: `signalValue`, `pValue`,
   `qValue` are expected on `mcols(peaks)` without checking. Add a
   defensive check before swapping ATAC sources.

4. **`--atac_files` is comma-separated**: better long-term to
   autodiscover via `list.files(opt$atac_dir, pattern = "\\.bb$",
   full.names = FALSE)` in the R script. Then you only edit `ATAC_DIR`.

5. **liftOver "first fragment per peak"** — fine for narrow ATAC, watch
   `n_split` count for broader peaksets.

6. **VCF chromosome naming**: prefers `<HYBRID>_het_snps.ucsc.vcf.gz`,
   falls back to `<HYBRID>_het_snps.vcf.gz` (Ensembl) with a warning
   that yields zero overlap. Always run `01_resources/03_convert_vcf_to_ucsc.sh`.

7. **No version pinning** for the `test_m` conda env. Add an
   `environment.yml`.

8. **The `0x_*` retry scripts have bumped limits in `config.sh`**
   (`RETRY_PARTITION=medium`, 72 h, 128 GB). Edit there, not in the
   scripts themselves.
